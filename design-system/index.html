Lending Design System

Getting started

Include the stylesheet imports in order. The system uses CSS custom properties for theming.

<!-- Required stylesheets (in order) --> <link rel="stylesheet" href="tokens/base.css"> <link rel="stylesheet" href="tokens/brand.css"> <link rel="stylesheet" href="tokens/semantic.css"> <link rel="stylesheet" href="tokens/component.css"> <!-- Component stylesheets --> <link rel="stylesheet" href="components/button.css"> <link rel="stylesheet" href="components/input.css"> <!-- ... additional components as needed -->
Theming

To use a different brand theme, replace brand.css with an alternative (e.g. brand-abn.css or brand-ing.css). All semantic tokens will automatically inherit the new brand values.

Dark mode

Add data-theme="dark" to the <html> element to enable dark mode. Semantic tokens include dark mode overrides.

Design tokens

Token architecture follows a three-tier system: base, brand, and semantic layers.

base.css
Primitive scales: spacing, typography sizes, neutral colours, status colours, radii, elevation.
brand.css
Brand identity: primary/secondary/accent colours, font families, shape preferences. Swappable for theming.
semantic.css
Intent-based aliases: --color-text-primary, --color-action-primary, etc. References base and brand tokens.
component.css
Component-scoped tokens: --button-radius, --input-border-color. References semantic layer.

  • Use the dark default for most app-level toasts — it feels like its own material and never gets lost on white UI
  • Only the accent stripe and icon change per variant; the card stays dark so all toasts feel like one family
  • Use --subtle only when the toast lives inside another surface (modal, settings panel)
  • Financial figures in messages use tabular figures via font-variant-numeric: tabular-nums
